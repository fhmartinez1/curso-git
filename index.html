Areas de GIT

Working Directory

El area de trabajo, basicamente, es la carpeta donde inicializamos el repositorio y donde vamos a estar creando, editando, eliminando los archivos de nuestro proyecto. Muy rara vez vamos a usar la linea de comandos en esta area.

  • dir || ls muestra el contenido de la carpeta
  • mk <name crea una carpeta en el directorio
  • cd <path abre un directorio
  • git init inicializa el repositorio de git
  • rm -rf .git elimina el repositorio

Staging Area (INDEX)

El area de control de cambios es aquella donde se realiza el seguimiento de los archivos y permite la comparacion del directorio de trabajo (Working Directory) con los commits almacenados en el historial de cambios. Cuando agregamos un archivo al index, se realiza la captura del codigo (Snapshot) y luego debemos confirmar si vamos a utilizarla o agregar mas modificaciones.

  • git status muestra al usuario el estado de los diferentes archivos que se encuentran en la carpeta del proyecto, dentro de los valores que podemos encontrar estan:
    • untracked: no se esta haciendo seguimieto, irrecuperable en caso de perdida.
    • staged: se realizo la captura, pero todavia no se confirmaron los cambios.
    • commited: los cambios fueron registrados en el historial mediante un commit.
    • modified: el archivo difiere con la ultima captura realizada, se esta haciendo seguimiento.
  • git add <file agrega el archivo al index.
  • git diff muestra las diferencias entre el working directoy y la ultima confirmacion.
  • git restore --staged <file quita la captura realizada del index.
  • git rm --catch <file remueve el archivo del area de control, deja de hacer seguimiento.
